## Developer Workflow

### Getting Started
```bash
git clone <repo>
cd photos
npm install
cp .env.example .env
# Edit .env with your DB credentials
npm start
# Visit http://localhost:8082
```

### Testing Changes
```bash
# After code changes:
npm test                    # Run Jest (once implemented)
curl -X GET http://localhost:8082/api/photos  # Quick API check
# Manual browser testing at http://localhost:8082
```

### Database Migrations
```bash
# After schema changes, create new migration:
# sql/migration_v4_to_v4.1_feature.sql
mysql -u root -p mydb < sql/migration_v4_to_v4.1_feature.sql
# Update DEPLOYMENT_CHECKLIST.md with new migration step
```

### Commit Convention
```
<type>: <subject>

<body (optional)>

feat:     Add new feature (image editing, playlist, etc.)
fix:      Fix bug (mobile PDF, caching, etc.)
refactor: Code reorganization (logging, error handling)
docs:     Documentation updates
test:     Test additions
chore:    Dependencies, build config
```
